Smart Hall Components

The Smart Hall system automates lecture hall operations including attendance tracking, environmental control, and safety features.

12V LED Strip
What it does: Provides lighting in the hall.

How it works:

  • Activated by a relay when motion is detected (via IR sensors)
  • Can also be triggered when the lecturer enters
  • Controlled by ESP32-CAM based on hall occupancy
DC Motor (Curtain Control)
What it does: Opens/closes curtains automatically.

How it works:

  • Requires an H-bridge or motor driver (e.g., L298N) to control direction
  • ESP32-CAM sends signals to the motor driver based on lecturer detection
  • Automatically closes when lecture ends or opens when lecturer arrives
KY-026 Flame Sensor
What it does: Detects fire via infrared (IR) light.

How it works:

  • Outputs a digital signal when flames are detected
  • Connected to an ESP32-CAM digital pin (e.g., GPIO2)
  • Triggers emergency protocol when fire is detected
Smoke Sensor (MQ-2)
What it does: Detects smoke/gas.

How it works:

  • Outputs an analog voltage proportional to smoke concentration
  • Connected to an ESP32-CAM analog pin (ADC)
  • Works in conjunction with flame sensor for comprehensive safety
Buzzer
What it does: Alarms during fire/smoke emergencies.

How it works:

  • Activated by ESP32-CAM via a digital pin (GPIO4)
  • Triggers when either flame or smoke sensor detects danger
  • Provides audible alert to evacuate the hall
ESP32-CAM
What it does: Central controller for the hall with face recognition capabilities.

How it works:

  • Face recognition for attendance using its built-in camera
  • Processes sensor data and controls relays
  • Runs a TensorFlow Lite model for face detection
  • Sends attendance logs to Firebase via Wi-Fi
  • Uses microSD card to store attendance logs temporarily if Wi-Fi is unstable
ESP32 Module
What it does: Central processing and communication hub.

Key Functions:

  • Sensor Interfacing: Connects to analog/digital sensors
  • Data Processing: Analyzes sensor data to trigger automated actions
  • Wireless Communication: Transmits data to Firebase and receives commands
  • Actuator Control: Drives high-power devices via relays
  • Power Management: Optimizes energy consumption
